Connect-Qualys

Connect to Qualys API and get back session $cookie for all other functions

Connect to Qualys API and get back session $cookie for all other functions.

Syntax 

Connect-Qualys [-qualysCred] <PSCredential> [-qualysServer] <String> [-assetTagging] [<CommonParameters>]

Parameters

Name Alias Description Required? Pipeline Input Default Value
-qualysCred use Get-Credential to create a PSCredential with the username and password of an account that has access to Qualys true false
-qualysServer FQDN of qualys server, see Qualys documentation, based on wich Qualys Platform you're in. true false
-assetTagging There are two different api endpoints, the new one is Asset Management and Tagging. Use this switch to get a cookie to make calls to Asset Management and Tagging false false False

Note

Author: Travis Sobeck, Kyle Weeks

Examples

EXAMPLE 1 
$cookie = Connect-Qualys -qualysCred $qualysCred -qualysServer $server
The variables $server is the qualys API Url that you wish to use. Will retreive a websession object that can be used in the $cookie variable in most other Qualys functions, unless the function specifies AssetTagging cookie.
EXAMPLE 2 
$cookie2 = Connect-Qualys -qualysCred $qualysCred -qualysServer $server -assetTagging
The variables $server is the qualys API Url that you wish to use. Will retreive a websession object using credentials that are base64 encoded. Which can be used in the $cookie variable in other Qualys functions that ask for an AssetTagging cookie.
